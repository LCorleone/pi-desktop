# Pi Desktop v1.4.0 — Thinking Display & macOS Fixes 🐛

A focused bug-fix release: garbled ANSI codes in thinking content, a thinking
label stuck on "Thinking…", and macOS session auto-naming silently failing.

> **Note:** This is a personal fork of [`gustavonline/pi-desktop`](https://github.com/gustavonline/pi-desktop).

---

## 🐛 Fixes

### Thinking content — ANSI escape codes stripped

pi emits 256-color SGR codes (e.g. `ESC[38;5;66m`) inside its thinking output.
These render as colors in a terminal, but in the web thinking panel the
non-printing ESC byte vanished and left visible garbage like
`[38;5;66mThinking:[39m` mixed into the text.

Thinking content is now stripped to clean, readable text. Fixed once in
`normalizeThinkingText` — the single function shared by both the message-path
and workflow-path thinking renders — so it's fixed everywhere at once.

### Workflow thinking label — now switches to "Thought"

Turns that included tool calls render thinking under a "workflow" panel. That
panel's label was hardcoded to `Thinking…` and never transitioned to `Thought`,
unlike the simple-message thinking panel which did. Every tool-bearing turn
looked stuck on "Thinking…" even after completion — on both macOS and Windows.

The label now derives from the same `thinkingAnimating` flag already used for
the icon and class, matching the message-path behavior exactly.

### macOS — session auto-naming now works

On macOS, session auto-naming silently failed and fell back to the first few
words of your message. Root cause: the standalone `pi --print` spawn didn't
inject `PATH`, so the `#!/usr/bin/env node` shebang couldn't resolve node when
pi is installed via nvm or Homebrew (GUI apps don't inherit shell PATH). The
error was `env: node: No such file or directory`.

It now mirrors the PATH injection the main RPC process already does. Windows
was unaffected — `pi.cmd` bypasses the shebang entirely.

### Thinking shimmer — stops correctly when streaming ends

The thinking animation and label now also require the message's `isStreaming`
flag, not just `isThinkingStreaming`. Prevents the shimmer from continuing in
edge cases after streaming has ended.

## 📦 Download

This fork builds **Windows** (`.exe`) and **macOS** (`.dmg`). Both triggered from a single workflow.

### ⚠️ Unsigned builds

**Windows:** Click **More info** → **Run anyway** on SmartScreen.

**macOS:** `xattr -cr "/Applications/Pi Desktop.app"` after mounting.
