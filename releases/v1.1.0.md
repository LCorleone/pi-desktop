# Pi Desktop v1.1.0 — Auto-Naming & Performance 🏷️

A substantial update to the Pi Desktop fork: rock-solid auto session naming,
smoother streaming in long sessions, and Windows UI polish.

> **Note:** This is a personal fork of [`gustavonline/pi-desktop`](https://github.com/gustavonline/pi-desktop).
> Full credit to the original authors.

---

## ✨ What's new

### 1. Auto session naming — works with every model

Sessions now get descriptive LLM-generated titles automatically. No more staring at "New session" tabs.

The title generator uses **pi itself** (`pi --print`) to process the request, so it inherits pi's credential discovery, proxy routing, TLS config, and API format handling. This means it works with **every model in the picker** — including providers behind corporate proxies or firewalls.

Highlights:
- ✅ Works with all providers (zai, deepseek, one_api, local proxies, etc.) — pi handles the API call, not manual HTTP
- ✅ Never clobbers a name you set manually or renames the wrong tab during async
- ✅ Sanitized output (no newlines, no markdown)
- ✅ Graceful fallback to first-5-words if anything fails
- ✅ Suppresses Windows console popup during title generation

### 2. Smoother streaming in long sessions

Fixed a progressive stutter that appeared as sessions grew long. Every streaming token used to trigger a full chat re-render. Renders are now coalesced to **one per animation frame** (~60fps) — streaming stays smooth regardless of session length.

### 3. Windows UI polish

- **JetBrains Mono** as the default app font
- Windows sidebar and chrome cleanup
- Dead workspace-tabs chrome removed
